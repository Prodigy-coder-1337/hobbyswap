# End-to-End User Flows

## 1. New user onboarding to first dashboard

1. User opens landing page.
2. Taps `Get Started`.
3. Completes sign-up form.
4. Completes onboarding quiz with hobbies, levels, availability, formats, and anonymity preference.
5. Sees the in-app guide.
6. Finishes guide and lands on the personalized dashboard with credit balance, next session, and intent-filtered recommendations.

## 2. Discover save-to-shortlist flow

1. Logged-in user opens `Swap`.
2. Switches between `All`, `Swaps`, `Teachers`, and `Workshops`.
3. Finds a listing in the map or list view.
4. Taps `Save`.
5. A shortlist confirmation modal appears instead of saving instantly.
6. User adds an optional note and confirms.

## 3. Equal swap contract flow

1. User taps the center `+` button.
2. Chooses `Swap`.
3. Selects a partner, what they will teach, what they want to learn, and the session structure.
4. Builds the availability grid.
5. Leaves `Equal swap` enabled.
6. Reviews a contract preview labeled `equal swap - no credits needed`.
7. Confirms and sees the new agreement in `Swap Log`.

## 4. Credit booking flow

1. User taps the center `+` button.
2. Chooses `Book session`.
3. Picks a teacher listing priced in credits.
4. Selects `Credits` as the payment method.
5. Reviews the booking breakdown.
6. Confirms the booking.
7. The learner sees held credits in the ledger until the session is marked complete.

## 5. Cash booking with escrow flow

1. User opens `Book session` from the center `+` flow.
2. Chooses a teacher or workshop with PHP pricing.
3. Selects `GCash`, `Maya`, `Card`, or `PayPal`.
4. Reviews the explicit 9% platform fee and teacher net payout.
5. Confirms the booking.
6. The session appears in `Swap Log` and the payout appears in the teacher tracker as held.
7. When the session is marked done, the payout moves to scheduled.

## 6. Teacher listing creation flow

1. User taps the sticky `Are you a teacher? Create a listing` nudge.
2. Lands in the `Create listing` mode of the center `+` flow.
3. Enters skill, level, format, and description.
4. Switches pricing between `Free`, `Credits`, `Cash`, or `Both`.
5. Watches the pricing inputs and fee explanation update dynamically.
6. Publishes the listing and sees it appear in discoverable results.

## 7. Session completion and review bonus flow

1. User opens `Weekly Challenges`.
2. Sees the progress bar, participant count, and credit reward.
3. Joins a challenge or logs progress on an existing one.
4. Later opens `Swap Log`.
5. Marks an active session done.
6. Leaves a 5-star review.
7. The teacher receives the review bonus and the credit ledger updates immediately.
