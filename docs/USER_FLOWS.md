# End-to-End User Flows

## 1. New user onboarding to first dashboard

1. User opens landing page.
2. Taps `Get Started`.
3. Completes sign-up form.
4. Completes onboarding quiz with hobbies, levels, availability, formats, and anonymity preference.
5. Sees the in-app guide.
6. Finishes guide and lands on the personalized dashboard with quick matches and local opportunities.

## 2. Marketplace purchase flow

1. Logged-in user opens `Swap`.
2. Filters listings and opens a detail sheet.
3. Taps `Buy now`.
4. App records a completed sale transaction.
5. Listing moves to completed state.
6. User later opens `History` and leaves a rating.

## 3. Skill/time swap contract flow

1. User opens `Contracts`.
2. Creates a new agreement with partner, teach/learn goals, sessions, format, and notes.
3. Contract appears in `Pending`.
4. Second party confirms terms.
5. Contract moves to `Active`.
6. Sessions are marked complete over time.
7. User opens `Swap Log` and sees progress reflected in hours and badges.

## 4. Discovery to consent-first messaging

1. User opens `Discover`.
2. Filters by hobby and distance.
3. Finds a nearby person and taps `Open`.
4. App creates a conversation request.
5. User opens `Messaging`.
6. Consent is granted.
7. User sends encrypted messages and uses a quick boundary script if needed.

## 5. Event hosting flow

1. User opens `Events`.
2. Switches to `Host`.
3. Fills out title, description, date, format, skill level, and capacity.
4. Submits the event for moderation review.
5. Event appears with `pending review` status.
6. Other users can later RSVP once the event is approved in the seeded workflow.

## 6. Resource borrowing flow

1. User opens `Resource Library`.
2. Searches for a lendable item.
3. Opens the policy sheet.
4. Accepts the damage/return terms.
5. Confirms reservation.
6. After use, returns to the same screen and taps `Return item`.

## 7. Weekly challenge collaboration flow

1. User opens `Weekly Challenges`.
2. Taps `Join challenge`.
3. Adds a partner from nearby users.
4. Submits a text/photo/video entry.
5. Another member votes anonymously.
6. Entry contributes to participation-based progress in the swap log.
